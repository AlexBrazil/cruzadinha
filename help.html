Configurar uma nova atividade

Use este passo a passo sempre que precisar montar uma cruzadinha nova a partir do arquivo data.json.

Passo a passo basico

  1. Faca uma copia de seguranca de data.json antes de editar.
  2. Abra o arquivo no editor de texto e ajuste os campos descritos abaixo.
  3. Salve e recarregue index.html no navegador para testar.

Campos essenciais

Cabecalho da atividade

  • title: nome mostrado no topo da pagina (ex.: "Cruzadinha - Capitais do Brasil").
  • taskDescription: texto introdutorio. Aceita HTML simples (paragrafos, listas, links).

Palavras (lista words)

Cada item representa uma pista e deve seguir o padrao abaixo:

{
  "clue": "Pergunta para o participante",
  "answer": "Resposta correta",
  "orientation": "across" ou "down",
  "fixWord": false,
  "row": null,
  "column": null,
  "extraImage": {
    "path": "./assets/images/imagem.png",
    "alt": "Descricao da imagem"
  }
}
  • clue (obrigatorio): texto da dica exibida na lista.
  • answer (obrigatorio): palavra correta. O sistema ignora maiusculas, acentos e espacos extras.
  • orientation (opcional): preferencia de posicao. Use "across" para horizontal ou "down" para vertical.
  • fixWord: deixe false para gerar automaticamente. Use true apenas se quiser travar a posicao.
  • row e column: coordenadas (base 1) usadas somente quando fixWord for true.
  • extraImage (opcional): define uma imagem clicavel de apoio. Informe path relativo a index.html e um alt descritivo.

Cadastre ao menos duas palavras e confirme que compartilham letras para permitir cruzamento.

Comportamento (behaviour)

  • enableSolutionsButton: controla o botao "Mostrar solucao".
  • enableRetry: ativa o botao "Tentar novamente".
  • Outros campos (enableInstantFeedback, scoreWords, applyPenalties, keepCorrectAnswers) podem ficar como no exemplo padrao.

Tempo (timer)

  • limitSeconds: tempo limite em segundos. Defina zero ou remova para deixar sem cronometro regressivo.

Visual (theme)

  • Cada cor (ex.: backgroundColor, gridColor, cellBackgroundColor) aceita o formato hexadecimal #RRGGBB.
  • Use esses campos para registrar o esquema de cores. Ajustes extras podem exigir mudancas em style.css.

Textos (l10n)

  • Atualize mensagens e rotulos exibidos na interface, como checkAnswer, tryAgain e extraClue.
  • Mantenha marcadores como @words ou @clue, pois sao substituidos automaticamente pelo sistema.

Acessibilidade (accessibility)

  • Personalize os textos lidos por leitores de tela (ex.: crosswordGrid, correct, solutionFor).
  • Preserve variaveis como @score e @solution para que a mensagem continue correta.

Boas praticas rapidas

  • Valide o JSON apos cada edicao usando um validador online ou abrindo index.html.
  • Confirme que imagens citadas em extraImage.path existem em ./assets/images.
  • Teste com respostas corretas e incorretas para verificar o comportamento dos botoes.
